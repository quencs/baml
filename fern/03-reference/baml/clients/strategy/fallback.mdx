---
title: fallback
---


You can use the `fallback` provider to add more resilience to your application.

A fallback will attempt to use the first client, and if it fails, it will try the second client, and so on.

<Tip>You can nest fallbacks inside of other fallbacks.</Tip>

```baml BAML
client<llm> SuperDuperClient {
  provider fallback
  options {
    strategy [
      ClientA
      ClientB
      ClientC
    ]
  }
}
```

## Options

<ParamField path="strategy" type="List[string]" required>
  The list of client names to try in order. Cannot be empty.
</ParamField>

### Timeout Options

Fallback clients support all standard timeout options plus `total_timeout_ms`:

- `connect_timeout_ms`: Max time to connect to each subclient
- `time_to_first_token_timeout_ms`: Max time to first token for each subclient
- `idle_timeout_ms`: Max time between chunks for each subclient
- `request_timeout_ms`: Max total time per subclient attempt
- `total_timeout_ms`: Max time for the entire fallback strategy

When subclients define their own timeouts, the **minimum (most restrictive) timeout wins**.

```baml
client<llm> MyFallback {
  provider fallback
  options {
    strategy [Primary, Backup, LastResort]

    // Per-subclient timeouts
    http_client {
      connect_timeout_ms 5000
      request_timeout_ms 30000
    }

    // Overall strategy timeout
    total_timeout_ms 120000  // Max 2 minutes for all attempts
  }
}
```

See [Timeout Configuration](/ref/llm-client-strategies/timeouts) for detailed documentation.

## retry_policy

Like any other client, you can specify a retry policy for the fallback client. See [retry_policy](retry-policy) for more information.

The retry policy will test the fallback itself, after the entire strategy has failed.

```baml BAML
client<llm> SuperDuperClient {
  provider fallback
  retry_policy MyRetryPolicy
  options {
    strategy [
      ClientA
      ClientB
      ClientC
    ]
  }
}
```

## Nesting multiple fallbacks

You can nest multiple fallbacks inside of each other. The fallbacks will just chain as you would expect.

```baml BAML
client<llm> SuperDuperClient {
  provider fallback
  options {
    strategy [
      ClientA
      ClientB
      ClientC
    ]
  }
}

client<llm> MegaClient {
  provider fallback
  options {
    strategy [
      SuperDuperClient
      ClientD
    ]
  }
}
```
