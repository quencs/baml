ARCHITECTURE HIGHLIGHTS:
- State: 40+ Jotai atoms (30+ core, 15+ derived)
- Single source of truth: runtimeInstanceAtom
- Navigation: 5-action decision tree heuristic
- Performance: O(1) function lookup, atomFamily for per-node state
- Immutable runtime pattern (recreate on changes, never mutate)

CRITICAL FILES:
1. /src/sdk/atoms/core.atoms.ts (638 lines)
   - All core state definitions
   - Selection atoms (function, test, workflow)
   - UI state (detail panel, layout, view mode)

2. /src/features/debug-panel/components/DebugPanel.tsx (349 lines)
   - Simulates IDE clicks on functions/tests
   - Load: sdk.diagnostics.getBAMLFiles()
   - Click handlers: handleFunctionClick, handleTestClick, handleTestRun
   - File operations: Add File, Edit Function buttons

3. /src/features/navigation/hooks/useCodeNavigation.ts (252 lines)
   - Listens to activeCodeClickAtom changes
   - Executes 5 navigation actions
   - Manages workflow switching and node selection
   - Handles async loading with timeouts

4. /src/sdk/navigationHeuristic.ts (250+ lines)
   - Decision tree algorithm
   - TEST click logic: find workflow or show empty state
   - FUNCTION click logic: 4-priority heuristic
     Priority 1: In current workflow → select-node
     Priority 2: In another workflow → switch-and-select
     Priority 3: Has tests → show-function-tests (LLM-only)
     Priority 4: No context → empty-state

5. /src/sdk/index.ts (600+ lines)
   - BAML SDK main class
   - Runtime factory pattern
   - File management API
   - Workflow and diagnostics APIs

STATE UPDATE FLOW (5 steps):
1. DebugPanel: setActiveCodeClick(CodeClickEvent)
2. Atom update: activeCodeClickAtom = event
3. Hook trigger: useCodeNavigation() effect fires
4. Decision: determineNavigationAction() returns action
5. Execution: Handlers update workflow/node/panel atoms

NAVIGATION DATA STRUCTURES:
- CodeClickEvent: { type: 'function'|'test', functionName, testName?, filePath, ... }
- NavigationAction: 5 types (switch-workflow, select-node, switch-and-select, show-function-tests, empty-state)
- BAMLFile: { path, functions: FunctionWithCallGraph[], tests: BAMLTest[] }

CRITICAL ATOMS FOR NAVIGATION:
1. activeCodeClickAtom - Stores latest click event
2. updateSelectionAtom - Shared update point (writes selectedFunctionNameAtom + selectedTestCaseNameAtom)
3. activeWorkflowIdAtom - Current workflow being viewed
4. selectedNodeIdAtom - Currently selected graph node
5. detailPanelAtom - Detail panel state (isOpen, position, activeTab)

PERFORMANCE OPTIMIZATIONS:
- atomFamily: Per-node state (nodeStateAtomFamily) - O(1) updates, no sibling renders
- Derived atoms: Cached until dependencies change
- allFunctionsMapAtom: Map-based O(1) function lookup instead of array search
- Debounced file updates: 50ms debounce on sdk.files.update()
- Immutable runtime: New instances on changes, no mutation
