# OpenAI Generic

The `openai-generic` provider supports any API using OpenAI's request/response format.

## Basic Example

```baml
client<llm> MyClient {
    provider "openai-generic"
    options {
        base_url "https://api.provider.com"
        model "<provider-specified-format>"
    }
}
```

## BAML-Specific Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `base_url` | string | `https://api.openai.com/v1` | API base URL |
| `api_key` | string | (none) | API key for Authorization header |
| `headers` | object | - | Additional request headers |

### Custom Headers

```baml
client<llm> MyClient {
    provider "openai-generic"
    options {
        base_url "https://api.provider.com"
        model "model-name"
        headers {
            "X-My-Header" "my-value"
        }
    }
}
```

## Reasoning Models

:::warning
For reasoning models (`o1`, `o1-mini`), use `max_completion_tokens` instead of `max_tokens`:

```baml
client<llm> ReasoningClient {
    provider "openai-generic"
    options {
        model "o4-mini"
        max_tokens null
    }
}
```
:::

## Provider Parameters

| Option | Type | Description |
|--------|------|-------------|
| `model` | string | Model identifier |
| `temperature` | float | Sampling temperature |
| `max_tokens` | int | Maximum response tokens |
| `top_p` | float | Nucleus sampling |

See the [OpenAI API documentation](https://platform.openai.com/docs/api-reference/chat/create) for all options.
