# Config

Configure BAML logging and environment settings via environment variables or programmatic API.

## Environment Variables

| Setting | Environment Variable | Description | Default |
|---------|---------------------|-------------|---------|
| Logging Level | `BAML_LOG` | Log level: `INFO`, `DEBUG`, `TRACE`, `WARN`, `ERROR`, `OFF` | `INFO` |
| JSON Mode | `BAML_LOG_JSON` | Log in JSON format: `1` or `0` | `0` |
| Max Message Length | `BAML_LOG_MAX_MESSAGE_LENGTH` | Max prompt/response log size (`0` for no limit) | `64000` |
| Color Mode | `BAML_LOG_COLOR_MODE` | Color output: `auto`, `always`, `never` | `auto` |

## Setting Environment Variables

Environment variables can be set in your shell, `.env` file, or programmatically:

```bash
# Shell
export BAML_LOG=DEBUG
export BAML_LOG_JSON=1

# .env file
BAML_LOG=DEBUG
BAML_LOG_JSON=1
```

Or pass environment variables via `with_options`:

<Tabs>
<TabItem value="python" label="Python">

```python
from baml_client import b

my_b = b.with_options(env={"BAML_LOG": "DEBUG"})
result = my_b.MyFunction("input")
```

</TabItem>
<TabItem value="typescript" label="TypeScript">

```typescript
import { b } from "baml_client"

const myB = b.withOptions({ env: { BAML_LOG: "DEBUG" } })
const result = await myB.MyFunction("input")
```
