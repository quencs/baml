# Multimodal Types

BAML supports multimodal types for working with images, audio, PDFs, and video.

## Overview

| Type | Description | URL Support | Base64 Support |
|------|-------------|-------------|----------------|
| `image` | Image data | Yes | Yes |
| `audio` | Audio data | Yes | Yes |
| `pdf` | PDF documents | No | Yes |
| `video` | Video data | Yes | Yes |

:::warning
You cannot use reserved type names as variable names (e.g., `image`, `audio`, `pdf`, `video`).
:::

## Image

### BAML Definition

```baml
function DescribeImage(myImg: image) -> string {
    client "openai/gpt-4o"
    prompt #"
        {{ _.role("user") }}
        Describe the image in four words:
        {{ myImg }}
    "#
}
```

### Host Language Usage

<Tabs>
<TabItem value="python" label="Python">

```python
from baml_py import Image
from baml_client import b

async def test_image_input():
    # From URL
    res = await b.DescribeImage(
        myImg=Image.from_url("https://example.com/image.png")
    )

    # From Base64
    image_b64 = "iVBORw0K...."
    res = await b.DescribeImage(
        myImg=Image.from_base64("image/png", image_b64)
    )
```

## PDF

:::note
PDF inputs must be provided as Base64 data or file path. URL-based inputs are not supported.
:::

### BAML Definition

```baml
function AnalyzePdf(myPdf: pdf) -> string {
    client "openai/gpt-4o"
    prompt #"
        {{ _.role("user") }}
        Summarize the main points of this PDF:
        {{ myPdf }}
    "#
}
```

### Host Language Usage

<Tabs>
<TabItem value="python" label="Python">

```python
from baml_py import Pdf
from baml_client import b

async def run():
    # From Base64
    b64 = "JVBERi0K...."

        myPdf=Pdf.from_base64("application/pdf", b64)

```

</TabItem>

## Video

### BAML Definition

```baml
function DescribeVideo(myVideo: video) -> string {
    client "openai/gpt-4o"
    prompt #"
        {{ _.role("user") }}
        Describe what happens in this video:
        {{ myVideo }}

```

:::note
When providing a video via URL, the URL is passed directly to the model. Some models cannot download external media; convert to Base64 first in those cases.

## URL Processing Configuration

Control how BAML processes media URLs before sending to providers:

```baml
client<llm> MyClient {
    provider anthropic
    options {
        media_url_handler {
            image "send_base64"   // Convert URLs to base64
            pdf "send_url"        // Keep URLs as-is
        }
    }
}
```

## Security Considerations

<details>
<summary>SSRF Attack Prevention</summary>

BAML's multimodal types make it easy to construct media from URLs. However, if you use untrusted user input, you may be vulnerable to [server-side request forgery (SSRF) attacks](https://portswigger.net/web-security/ssrf).

Attackers may be able to:
- Fetch files on your internal network
- Access external networks using your application's identity
- Drive up cloud network bandwidth costs

**Recommendations:**
- Only use URLs from trusted sources
- Validate URLs using allowlists or denylists
- Sanitize user input before constructing media objects

</details>
