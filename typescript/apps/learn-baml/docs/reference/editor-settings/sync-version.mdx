# baml.syncExtensionToGeneratorVersion

| Type | Default |
|------|---------|
| `"auto" \| "never" \| "always"` | `"auto"` |

## Values

| Value | Description |
|-------|-------------|
| `auto` | Sync extension version when mismatch detected |
| `never` | Never sync extension version |
| `always` | Always attempt to sync extension version |

When enabled, the extension downloads the correct version of `baml-cli` to generate client code, preventing version mismatch issues.

:::note Windows
On Windows platforms, the extension-sync feature is disabled when set to `auto`.
:::

## Usage

```json
{
  "baml.syncExtensionToGeneratorVersion": "auto"
}
```
